\begin{frame}{Проблема}

\begin{columns}[t]
\begin{column}{0.48\textwidth}
\textbf{Централизованные биржи (CEX)}
\begin{itemize}
    \item Регулятор может заблокировать оператора
    \item Single point of failure
    \item Закрытый matching --- нельзя проверить честность
    \item Требуют лицензий, KYC
\end{itemize}
\end{column}
\begin{column}{0.48\textwidth}
\textbf{HyperLiquid (эталон)}
\begin{itemize}
    \item Децентрализованный consensus (24 валидатора)
    \item Но \textbf{единый API Gateway} --- 2 outages в 2025
    \item \$360k/мес на инфраструктуру
    \item Закрытый код
\end{itemize}
\end{column}
\end{columns}

\vspace{0.8em}
\begin{block}{Рынок}
Perpetual DEX: \$2.3B+ ежедневного объёма (2026). HyperLiquid доминирует ($\sim$70\% market share). Единственный конкурент с децентрализованным matching.
\end{block}
\end{frame}

\begin{frame}{Решение}

\begin{columns}[t]
\begin{column}{0.48\textwidth}
\textbf{Децентрализация без компромиссов}
\begin{itemize}
    \item 3 валидатора, PBFT (2/3 голосов)
    \item Каждый валидатор = свой REST API
    \item Клиент сам выбирает, к кому подключаться
    \item При падении одного --- failover к другому
    \item Никакого API Gateway --- никакого SPOF
\end{itemize}
\end{column}
\begin{column}{0.48\textwidth}
\textbf{Ключевые преимущества}
\begin{itemize}
    \item[\textcolor{green}{\checkmark}] \textbf{Безопасность:} 2/3 подписей для вывода средств
    \item[\textcolor{green}{\checkmark}] \textbf{Open-source:} код публичный, любой может запустить ноду
    \item[\textcolor{green}{\checkmark}] \textbf{Экономия:} \$600/мес vs \$360k/мес (HyperLiquid)
    \item[\textcolor{green}{\checkmark}] \textbf{Регуляторная устойчивость:} \dq мы --- протокол, не биржа\dq
\end{itemize}
\end{column}
\end{columns}
\end{frame}

\begin{frame}{Telegram Mini App}

\textbf{Почему Telegram?}

\begin{itemize}
    \item 900M+ активных пользователей
    \item Встроенный кошелёк (Telegram Wallet --- TON USDT)
    \item WebApp SDK --- авторизация без паролей
    \item Мгновенные депозиты/выводы через TON
\end{itemize}

\textbf{Что сделано:}
\begin{itemize}
    \item UI: telegram.html + telegram-app.js (vanilla JS)
    \item Auth: Telegram WebApp initData (HMAC)
    \item Депозит USDT $\to$ торговля $\to$ вывод USDT
    \item Пользователи tg-* пропускают RSA-верификацию
\end{itemize}

\textbf{План:}
\begin{itemize}
    \item TON Connect / Tonkeeper интеграция
    \item Push-уведомления о сделках
    \item In-app графики
\end{itemize}
\end{frame}

\begin{frame}{Регуляторная стратегия}

\begin{enumerate}
    \item Incorporation: Cayman Islands / BVI
    \item Компания владеет IP, \textbf{но не управляет биржей}
    \item Сеть валидаторов --- открытая, код open-source
    \item Геоблокировка US + sanctioned countries (Cloudflare IP)
    \item \textbf{Без KYC} --- кошелёк = адрес, торговля без идентификации
    \item Ключевой аргумент: \dq мы --- протокол (L2 settlement layer), не биржа. Код открыт, валидаторы независимы\dq
\end{enumerate}

\vspace{0.5em}
\begin{block}{Пример}
HyperLiquid прошёл этот путь:\\
Cayman foundation + open-source + независимые валидаторы = SEC не может заблокировать протокол.
\end{block}
